# 待决问题

## 概述

本文档记录 Spring 生态学习项目开发过程中的未决设计问题和已做出的决策。

## 未决问题

| # | 问题 | 上下文 | 候选方案 | 状态 |
|---|------|--------|----------|------|
| 1 | 示例项目版本统一升级策略 | 当前部分示例使用 Spring Boot 3.5.0 + Java 17，目标为 4.0.5 + Java 21 | A: 一次性全部升级 / B: 分批逐步升级 | 待讨论 |
| 2 | Spring Security 示例是否需要同时覆盖 OAuth2 和 JWT | JWT 更通用，OAuth2 更贴合企业场景 | A: 只做 JWT / B: 两个都做 / C: JWT 为主，OAuth2 为扩展 | 待讨论 |
| 3 | MyBatis 示例是单独项目还是合并到 MySQL 示例 | 当前 MySQL 示例使用 JPA，MyBatis 作为替代方案 | A: 合并到同一个项目 / B: 独立项目对比两种方案 | 待讨论 |
| 4 | 微服务示例项目应采用单仓库多模块还是独立仓库 | Spring Cloud 微服务通常多仓库部署 | A: Monorepo 多模块(maven multi-module) / B: 独立项目通过 docker-compose 编排 | 待讨论 |
| 5 | 是否需要引入 AI/LLM 集成主题 | Spring AI 1.0 已发布，是热点方向 | A: 新增 AI 集成主题 / B: 暂不引入，保持传统生态 | 待讨论 |
| 6 | 文档国际化策略 | 当前部分文档中英混合，需确定国际化标准 | A: 每个文件内中英对照 / B: 独立的中文/英文文档目录 | 待讨论 |

## 已决策

| # | 问题 | 决策 | 理由 | 日期 |
|---|------|------|------|------|
| 1 | 项目结构 | Monorepo：根项目为文档 + examples/ 为独立子项目 | 便于统一管理版本和依赖，降低新人上手成本 | 2025-03 |
| 2 | 构建工具 | Maven（非 Gradle） | 国内 Java 生态 Maven 覆盖率更高，降低学习门槛 | 2025-03 |
| 3 | 中间件管理方式 | Docker Compose | Docker 保证环境一致性，避免本地安装各中间件 | 2025-03 |
| 4 | 示例项目独立运行 | 每个示例项目为独立 Maven 项目，无交叉依赖 | 确保每个示例可单独运行学习，不被其他项目打断 | 2025-03 |
| 5 | 许可证 | MIT | 最宽松的许可证，最大化传播 | 2025-03 |
| 6 | Java 版本 | Java 21 (LTS) | Spring Cloud Alibaba 2025.1 最低要求，也是最新 LTS | 2025-03 |
